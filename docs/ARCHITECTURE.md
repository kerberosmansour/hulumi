## Overview

Hulumi is an Apache-2.0 TypeScript monorepo that ships hardened-by-default Pulumi components for AWS, a CrossGuard policy pack, a local-first drift classifier, and a Claude Code skill. Today the entire IaC surface targets AWS only тАФ the GitHub variant is in flight under [`docs/RUNBOOK-hulumi-github.md`](./RUNBOOK-hulumi-github.md). All three publishable npm packages (`@hulumi/baseline`, `@hulumi/policies`, `@hulumi/drift`) are at version 1.0.0 and ship with SLSA Build L3 attestation.

## Entry Points

- **`/hulumi-threat-model <scenario-id>`** тАФ Claude Code skill (lives at `~/.claude/skills/hulumi-threat-model/` after install). Five prebuilt AWS scenarios. Writes structured threat-model markdown into the user's working directory.
- **`pulumi up`** in a user's program that imports `@hulumi/baseline` тАФ the components register child resources with hardened defaults.
- **`PolicyPack`** loaded via `pulumi policy publish` or evaluated locally тАФ `HulumiHardeningPack` and `CisV5Pack` enforce invariants at preview-time.
- **`DriftClassifier.classify()`** тАФ invoked from a user's drift-detection wrapper; uses Pulumi Automation API + CloudTrail + git log + provider version as the four signal sources.
- **`pnpm run lint:license-boundary`** тАФ repo-side license-boundary lint, also runs in CI.
- **`pnpm run lint:exact-pin-guard`** тАФ repo-side exact-pin guard, also runs in CI.

## Test Architecture

Three distinct test surfaces, all Vitest 1.6.1:

| Layer        | Where                                                                          | How to run                                                                            |
| ------------ | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------- |
| Unit / BDD   | `packages/<pkg>/tests/*.test.ts`, `packages/<pkg>/tests/setup.ts`              | `pnpm -r test` (build required first тАФ examples import from `dist/`)                  |
| Integration  | `packages/<pkg>/tests/integration/*.integration.test.ts`                       | `HULUMI_INTEGRATION=1 pnpm test:integration` (real AWS sandbox; weekly CI by default) |
| Repo-wide    | `tests/skill-bdd/*.test.ts`                                                    | included in `pnpm -r test`                                                            |

Mock-runtime BDD pattern: `pulumi.runtime.setMocks()` per package; tests use a `settlePulumi()` helper (40 `setImmediate` ticks) тАФ see [`packages/baseline/tests/setup.ts`](../packages/baseline/tests/setup.ts).

TLA+-alignment meta-test: [`packages/drift/tests/tla-alignment.test.ts`](../packages/drift/tests/tla-alignment.test.ts) enforces lockstep between `packages/drift/src/verdict.ts` and `docs/TLAdocs/hulumi/HulumiDrift.tla`.

Forbidden-shortcut lints-as-tests:

- `packages/drift/tests/no-shell-exec.test.ts` тАФ bans `child_process` from `packages/*/src/`.
- `packages/drift/tests/no-sleep.test.ts` тАФ bans `setTimeout` / `sleep` outside sanctioned probe paths.
- `scripts/license-boundary-lint.mjs` (run via `pnpm run lint:license-boundary`) тАФ bans verbatim framework-prose embedding.
- `scripts/exact-pin-guard.mjs` (run via `pnpm run lint:exact-pin-guard`) тАФ bans `@pulumi/*` version drift off integrity-hash pins.

CI ([`.github/workflows/ci.yml`](../.github/workflows/ci.yml)) runs all of the above plus the cooling-off diff, attestation dry-run, and DCO `Signed-off-by` check.

## Constraints in force at HEAD

These are documented inline so future agents can see them without traversing every file:

- Node тЙе 20.0.0, pnpm тЙе 9.0.0, packageManager pinned to `pnpm@9.12.0`.
- `@pulumi/pulumi@3.232.0`, `@pulumi/aws@7.27.0`, `@pulumi/policy@1.20.0` exact-pinned with integrity hashes; bumps go through 72h/24h cooling-off CI gate.
- Apache-2.0 throughout. No verbatim CCM / AICM / CAIQ / CIS Benchmark / NIST control text in source тАФ `license-boundary-lint` blocks PRs.
- SLSA Build L3 on every npm release; npm trusted publishing via OIDC; no `NPM_TOKEN`.
- DCO sign-off required on every commit (CI-enforced).
- No telemetry, no hosted-service runtime dep, no shell-exec in `packages/*/src/`, no `eval`, no `setTimeout`/sleep outside sanctioned probe paths (test-enforced).
- Atomic three-package release: `@hulumi/baseline`, `@hulumi/policies`, `@hulumi/drift` ship the same version on the same day.
- Mandatory IaC tag: `hulumi:iac-role=true` on IaC execution roles at v1.0.
- `pulumi.dynamic.Resource` does NOT work under vitest's worker pool тАФ use `dependsOn` instead. Documented gotcha; escape hatch at [`packages/baseline/src/aws/probes/poll.ts`](../packages/baseline/src/aws/probes/poll.ts).
- `pnpm -r build` MUST run before `pnpm -r test` тАФ example tests import from `dist/` via the `exports` map.
