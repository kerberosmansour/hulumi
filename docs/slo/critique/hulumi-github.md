## Threat-model input adaptation (read first)

The Hulumi pattern instead treats the `/hulumi-threat-model` skill output (M1's four exemplars at `docs/threat-model-examples/github-*.md`) plus the per-milestone abuse-case rows (`tm-hulumi-github-abuse-N` IDs embedded in each milestone's BDD table) as the threat model.

The Security pass below uses those abuse-case row IDs as threat-model row citations. **One Security finding (S4 — V17 missing threat model) explicitly raises this as a structural concern** and proposes shipping a consolidated `docs/design/hulumi-github-threat-model.md` in M3 or M5. The user can resolve by either (a) accepting the adaptation as documented or (b) running `/slo-architect` to produce the formal doc before `/slo-execute` begins.
