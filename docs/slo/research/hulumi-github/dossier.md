# Research Dossier — Hulumi for GitHub

> Authoritative companion: [`raw.md`](./raw.md) (58 KB, generated by `sldo-research` 2026-04-26 over 4 iterations / 14 web searches). This file is the slo-research-prescribed structured slice; `raw.md` is what `/slo-plan` consumes.

## Market

Same wedge persona as Hulumi-for-AWS: platform engineers at companies adopting opinionated IaC and accountable for security posture across cloud + GitHub. No public market-sizing exists for "GitHub-as-IaC hardening tooling" specifically — proxies in 2026:

- **GitHub Enterprise Cloud sticker** — $231/user/year (BlockSentient 2026 review), the tier required for the audit-log REST API.
- **AGPL-3.0 commercial-pressure signal** — `step-security/secure-repo` (1,408 commits, 32 releases, AGPL-3.0) creates demand for Apache-2.0-licensed equivalents that enterprises can vendor.
- **Vendor / foundation maintenance signal** — `github/safe-settings` (1,202 commits on `main-enterprise`, GitHub-published) and `ossf/allstar` v4.5 (724 commits, OpenSSF-maintained) indicate sustained foundation-level investment in adjacent shapes.
- **Incident pressure** — UNC6426 (March 2026, ~500 vulnerable role ARNs across 275 AWS accounts), trivy-action March 2026 (75/76 tags compromised), tj-actions/changed-files (23k repos), Sysdig Shai-Hulud worm (Nov 2025), OpenAI Codex token-stealing (patched Feb 2026), Vercel April 2026 — sustained incident frequency around the four threat-model scenarios this wedge would headline.

Direct downloads-as-spend proxy via npm `@pulumi/github` and Terraform Registry `cloudposse/terraform-github-repository` was not run in this dossier.

## Technical prior art

- **`pulumi-github` v6.13.0 (2026-04-24, Apache-2.0)** — mandatory baseline. First-class IaC bindings for `RepositoryRuleset`, `OrganizationRuleset`, `OrganizationSettings`, `EnterpriseSecurityAnalysisSettings`, `ActionsOrganizationPermissions`, `ActionsOrganizationOidcSubjectClaimCustomizationTemplate`. Source: <https://github.com/pulumi/pulumi-github>.
- **`terraform-provider-github` v6.12.0 (2026-04-23, MPL-2.0)** — parity reference for resource-list completeness. Same two structural gaps (Code Security Configurations attach, audit-log streams) confirms platform-wide.
- **GitHub Well-Architected SSDF mapping** — the cleanest infra-vs-workflow split for 2026: PO.2/PO.3/PO.4/PO.5/PS.1/PW.4/PW.5/PW.6/PW.7/RV.1 = Infrastructure-as-Code; PS.2/PW.1/PW.2/PW.8/RV.2/RV.3 = Workflow. <https://wellarchitected.github.com/library/scenarios/nist-ssdf-implementation/>.
- **NIST SP 800-218 v1.1 (2022-02-03 final, public domain)** — canonical SSDF mapping baseline. SP 800-218r1 IPD (v1.2) closed comments 2026-01-30 with no second draft published as of 2026-04-25.
- **SLSA Source Track v1.2** — L1–L3 are repo/org configuration; L4 mixes config with per-PR review behavior. <https://slsa.dev/spec/v1.2/source-requirements>.
- **OpenSSF Scorecard** — canonical configuration-driven-vs-analysis-driven check taxonomy: `Branch-Protection`, `Pinned-Dependencies`, `Token-Permissions`, `Webhooks` (configuration-driven) vs `Code-Review`, `Maintained`, `SAST`, `Vulnerabilities` (analysis-driven).
- **CIS GitHub Benchmark v1.2.0 (March 2026 cohort, CC BY-NC-SA 4.0)** — framework-ID source for `cis-github.ts` mappings. Section numbering gated behind CIS WorkBench member access.
- **MITRE ATT&CK T1195 (Supply Chain Compromise) + sub-techniques** — the only ATT&CK anchor for SCM-specific threats. No SCM-specific matrix exists in ATT&CK as of 2026; threat-model binding is lossy.
- **GitHub Webhook Events catalog** — push-model audit-log fallback for non-GHEC tiers: confirmed delivery of `branch_protection_rule`, `repository_ruleset`, `secret_scanning_alert`, `dependabot_alert`, `code_scanning_alert` (private repos: GHAS-licensed only), `member`, and org-only `organization`. <https://docs.github.com/en/webhooks/webhook-events-and-payloads>.

## Regulatory / legal

- **CIS Benchmarks license** — CC BY-NC-SA 4.0 plus CIS Terms of Use for Non-Member Products forbid posting, sublicensing, and commercial redistribution. Apache-2.0 outbound stays compatible iff Hulumi cites by ID and upstream URL only and never embeds control text. **Same posture as existing CIS AWS Foundations Benchmark mapping at `packages/baseline/src/mappings/cis-aws.ts` — no divergence.**
- **`step-security/secure-repo` is AGPL-3.0** — Apache-2.0 Hulumi packages cannot vendor any portion. Cite as canonical example of the workflow-side-of-the-boundary; do not vendor.
- **GitHub enterprise audit-log REST API auth-mode constraint** — endpoints for `GET /enterprises/{enterprise}/audit-log` and the `…/audit-log/streams/*` family accept _only_ classic PATs carrying `read:audit_log`. GitHub App installation tokens, GitHub App user tokens, fine-grained PATs, and OAuth-app tokens are all rejected by GitHub-the-vendor's documented constraint.
- **GitHub Advanced Security license** — private-repo `code_scanning_alert` webhook delivery requires GHAS. Drift-classifier verdict must distinguish `feature-not-licensed` from `no-drift`.
- **NIST SSDF v1.2 final timing** — IPD comment period closed 2026-01-30; v1.2 not final as of 2026-04-25. Mappings target v1.1 to avoid renumbering risk if NIST publishes between now and Hulumi-for-GitHub ship.
- **GitHub flat `*_enabled_for_new_repositories` fields are marked "endpoint closing down notice. Please use code security configurations instead."** Deprecation timeline is not yet sourced — risk to `OrganizationSettings`-based components at all tiers.

## Open questions that research did not answer

- **Wedge persona's GitHub plan tier (GHEC, Team, or Pro)** — the brief did not specify; determines Option A vs Option B as default architecture. **Highest-priority blocker for `/slo-plan`.**
- **CIS GitHub Benchmark v1.2.0 section numbering** — gated behind CIS WorkBench member access. Public proxies do not work (`aquasecurity/chain-bench` covers the CIS Software Supply Chain Guide, not GitHub Benchmark; `M-Davies/cis-github-benchmark` self-describes as not-yet-existing). Blocks `mappings/cis-github.ts` authoring with confident IDs until WorkBench access is obtained.
- **Deprecation timeline for `PATCH /orgs/{org}` flat `*_enabled_for_new_repositories` fields** — GitHub's "endpoint closing down" notice has no published date.
- **Empirical demand survey for threat-model scenarios** — `d > c > f > e` ordering is incident-anchored, not survey-anchored. Snyk State of Software Supply Chain 2026 / GitHub Octoverse 2026 / practitioner-survey corroboration would tighten the ranking.
- **`terraform-provider-github` v6.12.0 vs `pulumi-github` v6.13.0 exhaustive property-by-property diff** — observed two structural gaps in common but not run resource-by-resource. A latent Terraform-only resource would weaken Recommendation 1.
- **`repository_ruleset` webhook delivery on Team tier** — checked at docs level, not against a live non-GHEC org. Closes residual risk in Option B's webhook fallback.
- **OIDC `job_workflow_ref` template ergonomics for N×M workflow×env matrices** — recommended three-axis default works for 1×1 patterns; 2+×2+ needs a sourced exemption-cookbook strategy rather than an opinion.
- **Pulumi provider docs do not endorse `appAuth` as the recommended IaC role** — the "use a GitHub App" stance is Hulumi-owned, not Pulumi-derived. If Pulumi later publishes a fine-grained-PAT-first opinion, Hulumi guidance may need to re-anchor.
- **`pulumi-github` v6.13.0 release date precision** — one WebFetch slice returned a "2024" string; surrounding evidence supports 2026-04-24 but contradiction not fully resolved.
- **MITRE ATT&CK has no SCM-specific matrix in 2026** — T1195 anchor is the canonical fallback; impact of lossy ATT&CK binding on the wedge audience is unconfirmed.
- **Auth-mode constraint on audit-log streaming-management endpoints** — sourced explicitly for `/enterprises/{enterprise}/audit-log` but the streaming-specific docs page in iter-2 did not requote the constraint. Confidence remains high (uniform REST docs) but a recheck before locking design is warranted.
