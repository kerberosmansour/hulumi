# Milestone 3 — `@hulumi/policies.github.HulumiGithubHardeningPack` H1–H4 + `CisGithubV1Pack` (IDs-only mapping) + `G_OIDC_1` rule

Parent runbook: [docs/RUNBOOK-hulumi-github.md](../RUNBOOK-hulumi-github.md). Read the runbook's Global Execution Rules (especially **Rule 0 — the infra-only scope contract**) + Global Entry Rules + [docs/lessons/hulumi-github-m1.md](../lessons/hulumi-github-m1.md) + [docs/lessons/hulumi-github-m2.md](../lessons/hulumi-github-m2.md) before starting.

**Goal**: After M3, two new CrossGuard `PolicyPack` modules ship under `@hulumi/policies/github`: `HulumiGithubHardeningPack` enforcing four hardening invariants (H1: no raw `github.Repository`; H2: no `OrgFoundation` with wildcard custom OIDC template; H3: no AWS / Azure / GCP IAM trust policy with `StringLike` or wildcard `sub`; H4: tier monotonicity — Startup-Hardened MUST emit strictly more controls than Sandbox), and `CisGithubV1Pack` carrying CIS GitHub Benchmark v1.2.0 IDs-only mappings (with explicit "section numbers gated behind WorkBench access — see follow-up issue" placeholders that the lint rejects as TODOs only if they ship to a release tag). The standalone `G_OIDC_1` rule lifts H3 into a reusable module so other packs can compose it. M3 also adds the deferred `hulumi:controls` tag to `SecureRepository` (M1) and `OrgFoundation` (M2) outputs as the staged-migration completion.

**Context**: Research [synthesis §"OIDC misconfiguration is the single highest-impact threat-model scenario"](../research/hulumi-github/synthesis.md) anchors `G_OIDC_1` in UNC6426 (March 2026, ~500 vulnerable role ARNs across 275 AWS accounts) and the AWS / Azure / GCP / GitHub three-axis convergence. CIS license posture carries over from `cis-aws.ts` unchanged (CC BY-NC-SA 4.0 + Non-Member Terms forbid posting/sublicensing/commercial redistribution; ID-only citation is Apache-2.0 compatible). The `cis-github.ts` mapping ships **incomplete** by design — open question #2 in [`docs/research/hulumi-github/dossier.md`](../research/hulumi-github/dossier.md) flags WorkBench-membership-gated section numbering. The IDs-only `as const` table is structured so each CIS section has a placeholder array, ready to be filled in once WorkBench access is obtained. License-boundary-lint refuses any commit that embeds verbatim CIS GitHub Benchmark text. Tier monotonicity (H4) inherits the AWS-side invariant verbatim — at AST level, the set of `hulumi:controls` IDs emitted by a Startup-Hardened component is a strict superset of the Sandbox set.

**Important design rule**: `HulumiGithubHardeningPack` and `CisGithubV1Pack` are **separate `PolicyPack` modules** — the AWS-side pattern at [`packages/policies/src/aws/packs/hulumi-hardening.ts`](../../packages/policies/src/aws/packs/hulumi-hardening.ts) and [`packs/cis-v5.ts`](../../packages/policies/src/aws/packs/cis-v5.ts) demonstrates the one-PolicyPack-per-process invariant. Users select packs via `pulumi policy publish` per pack. `G_OIDC_1` is exported as a standalone rule from `@hulumi/policies/github` so users (and `HulumiGithubHardeningPack`) can compose it. The `hulumi:controls` tag on `SecureRepository` and `OrgFoundation` outputs is added in this milestone via additive edits to `packages/baseline/src/github/secure-repository.ts` and `org-foundation.ts` plus their sub-components — this is the deferred staged-migration from M1 / M2 explicitly recorded in those milestones' lessons files.

**Refactor budget**: `Surgical addition + targeted additive edit.` New `packages/policies/src/github/` directory (mirrors `packages/policies/src/aws/`), new `packages/baseline/src/mappings/{cis-github,nist-ssdf-v1.1}.ts` (IDs-only). Targeted additive edits to `packages/baseline/src/github/secure-repository.ts`, `org-foundation.ts`, and the four `org-*.ts` sub-components to add `hulumi:controls` tags using the new mappings. `packages/policies/src/index.ts` and `packages/baseline/src/mappings/index.ts` are extended.

## Notes

- This milestone is the **mid-runbook centerpiece**: M1 + M2 ship the components, M3 ships the enforcement, M4 ships the drift loop. If M3's enforcement is not strict by default, the rest of the runbook fails to deliver the Hulumi promise.
- The `cis-github.ts` mapping is **deliberately incomplete** until WorkBench access is secured. License-boundary-lint extension MUST reject TODO-WORKBENCH placeholders on `release-*` git tags so v1.x.0 does not ship with placeholders.
- G_OIDC_1's three-cloud coverage (AWS / Azure / GCP) is locked from M3. New clouds (e.g. Oracle Cloud, IBM Cloud) are v1.1+ extensions — not silent additions.
- The `hulumi:controls` tag staged-migration is now complete. Future components (e.g. v1.1 audit-log adapter) ship the tag from day one.
- M4's drift adapter relies on the AST-level controls-vs-mapping-table cross-check landing here; without it, drift detection cannot distinguish "a real change" from "a `hulumi:controls` tag value drift." M4's first regression-protection test imports `cisGithub` and `nistSsdfV11` to verify the tag-value invariant.
