# Ticket 139: Migrate SecureBucket off Deprecated S3 V2 Classes

## GitHub Issue

- Issue: <https://github.com/kerberosmansour/hulumi/issues/139>
- Title: `Migrate SecureBucket away from deprecated Pulumi AWS S3 V2 resource classes`
- Workpad comment: <https://github.com/kerberosmansour/hulumi/issues/139#issuecomment-4464400107>

## Contract Block

| Field            | Contract                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Problem          | `SecureBucket` still constructs deprecated Pulumi AWS S3 V2 classes (`BucketV2`, `BucketServerSideEncryptionConfigurationV2`, `BucketVersioningV2`, `BucketObjectLockConfigurationV2`, `BucketLoggingV2`, `BucketLifecycleConfigurationV2`). New deployments emit warning noise and future provider changes become harder to reason about.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Outcome          | New `SecureBucket` deployments construct current non-V2 S3 classes where Pulumi AWS 7.27.0 exposes equivalents, while preserving hardening defaults and adding aliases back to the old V2 type tokens for existing stacks. Policies and drift tests continue to accept both V2 and non-V2 bucket tokens during migration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Non-goals        | Do not change package versions, dependency pins, IAM/control mappings, tier semantics, tag semantics, policy IDs, drift execution behavior, or release workflow. Do not remove compatibility for existing V2 state tokens.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Scope allow-list | `packages/baseline/src/aws/secure-bucket.ts`; `packages/baseline/src/aws/secure-bucket.args.ts`; `packages/baseline/src/aws/secure-bucket.outputs.ts`; `packages/baseline/tests/secure-bucket.test.ts`; `packages/baseline/tests/account-foundation.test.ts`; `examples/secure-bucket-smoke/tests/smoke.test.ts`; `packages/policies/src/aws/hulumi-hardening-pack.ts`; `packages/policies/src/aws/cis-v5-pack.rules.ts`; `packages/policies/tests/hulumi-hardening-pack.test.ts`; `packages/policies/tests/cis-v5-pack.test.ts`; `packages/policies/README.md`; `packages/drift/tests/discovery.test.ts`; `packages/drift/tests/reconciler.test.ts`; `packages/drift/tests/s3-sweeper.test.ts`; `packages/drift/README.md`; `docs/components/secure-bucket.md`; `docs/tiers.md`; `docs/faq.md`; `docs/v2-migration.md`; `docs/cookbooks/migration-mid-stack-adoption.md`; `docs/cookbooks/policy-pack-rollout.md`; this ticket file. |
| Compatibility    | Existing stacks must use Pulumi aliases for the old child resource type tokens. `SecureBucket.bucket` changes to the current `aws.s3.Bucket` class; documented as a TypeScript type-surface change, with unchanged practical outputs (`id`, `arn`, `bucketDomainName`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Verification     | Add BDD tests before implementation. Run baseline focused tests/typecheck/build, policy focused tests/typecheck/build, drift focused tests/typecheck/build, format, license-boundary, and diff checks.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |

## BDD Scenarios

| Scenario                                         | Expected red state                                         | Expected green state                                                                                                                          |
| ------------------------------------------------ | ---------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| SecureBucket emits current S3 resource tokens    | Existing SecureBucket tests expect V2 tokens.              | SecureBucket tests expect non-V2 tokens for bucket, SSE, versioning, object lock, logging, and lifecycle.                                     |
| Existing stacks have alias migration path        | No source-level guard confirms old V2 aliases are present. | Test verifies `secure-bucket.ts` contains aliases for all migrated V2 type tokens.                                                            |
| Policy compatibility accepts both token families | H1/H4 managed-child logic only recognizes V2 child URNs.   | H1/H4 recognize SecureBucket-managed children for both `aws:s3/bucket:Bucket` and `aws:s3/bucketV2:BucketV2`; H4 accepts both logging tokens. |
| Drift compatibility accepts both token families  | Drift tests only exercise V2 S3 tokens.                    | Drift discovery/reconciler/sweeper tests exercise non-V2 bucket tokens while preserving V2 coverage.                                          |
