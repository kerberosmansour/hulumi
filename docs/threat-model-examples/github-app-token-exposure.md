# Threat Model — GitHub App / installation-token exposure

> Exemplar handcrafted 2026-04-26. Generated by the `/hulumi-threat-model` Claude Code skill.

## Scenario

A platform engineer adopts GitHub App authentication for IaC (the recommended Hulumi default for the `pulumi-github` provider). Recent incidents — OpenAI Codex token-stealing branch-name injection (patched February 2026) and the Vercel April 2026 compromise — demonstrate that App installation tokens with broad org-admin permissions create blast-radius equivalent to (or worse than) classic PATs.

## Actors

- Platform Engineer (trusted, installs the IaC GitHub App)
- GitHub App Maintainer (trusted; could be compromised — Vercel April 2026 pattern)
- External Attacker (untrusted, exfils a leaked installation token)
- Compromised CI runner (semi-trusted)

## Assets

- GitHub App installation token (short-lived, ~1h TTL by default)
- GitHub App private key (long-lived; manufactures installation tokens)
- App permissions surface (which org-level / repo-level scopes the App has)
- Pulumi state and secrets that the App-authenticated `pulumi up` writes / reads

## Threats (STRIDE)

| Type | Name | Description | Controls |
|---|---|---|---|
| I | Installation token leaked via runner logs or stack traces | An error log emits an HTTP error response that includes a partial token fragment. | CCM:IVS-09, NIST-SSDF-v1.1:PW.7, OpenSSF-Scorecard:Token-Permissions, MITRE-ATTCK:T1195, GitHub-Well-Architected:RV.1 |
| T | App private-key compromise → arbitrary installation tokens | Private key exfiltrated; attacker manufactures arbitrary installation tokens for any installed org. | CCM:CEK-04, NIST-SSDF-v1.1:PO.5, GitHub-Well-Architected:PO.5, MITRE-ATTCK:T1195, CIS-GitHub-v1.2.0 |
| E | Over-broad App permissions → privilege escalation on token leak | App installed with `org-admin: write` for one-time bootstrap; permissions never trimmed after. | CCM:IAM-07, NIST-SSDF-v1.1:PO.3, OpenSSF-Scorecard:Token-Permissions, GitHub-Well-Architected:PO.3 |
| S | App impersonation via OIDC misconfiguration | Wildcard `sub` (UNC6426) lets an attacker-controlled workflow mint IaC-looking tokens. | CCM:IAM-10, NIST-SSDF-v1.1:PO.5, GitHub-Well-Architected:PO.3, MITRE-ATTCK:T1195 |

- `@hulumi/baseline.github.OrgFoundation` — Shipped in Hulumi v1.1.0 M2. Disables classic-PAT auth at startup-hardened tier; ships token-redaction layer in audit-event emitter (per critique S2).
